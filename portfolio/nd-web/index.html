Newsday Website

Newsday September 2017

The latest redesign of Newsday LLC launched in January 2018. In this version we made drastic changes and improvements to the UX, improving usability and utilizing modern frameworks to deliver a robust experience unique to the news industry.

Design Process

As the senior designer on this project, I was an intergal part of it's conception. I provided mockups from wireframes to full-size comps. I used HTML, SASS and JQuery to build mobile-first, responsive front-end templates that were integrated into a VSCode framework.

Objectives

  • Enhancing the company's online presence with the market's wayning demand for print journalism.
  • How users currently interact with the website and they're expectations for improvement.
  • Enhancing the user experience with the use of new technology and the technical challenges they can pose.

User Experience

My approach as front-end developer is by putting the user first. Having a wide range understanding of the following can assist in achieving strong user experience:

Analyzing user behaviour has been an important part of improving our user's experience. Tools such as Google Analytics and Hotjar have given insights into users habits, so we can learn about how they behave and interact with the Newsday.com website.

The website is continuously updated with a vast amount of changes since its initial launch. Our working process involves use of Jira for code management.

Responsive Design

Our working process involves the use of planned tests as well as release-specific tests that are carried out by the developers. Then by a quality assurance team. We are working towards expanding our automated testing with tools such as RSpec on the back-end and Jasmine on the front-end.
